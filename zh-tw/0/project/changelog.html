Version 0.28.0#

July 4, 2025

General#

  • Update Upgraded to Python 3.13.2. #5498

  • ABI Break Upgraded Emscripten to 4.0.9 #5343 #5350 #5357 #5334 #5363 #5360 #5379 #5382 #5333 #5391 #5397 #5337 #5399 #5401 #5403 #5332 #5557 #5560 #5595 #5616 #5653

  • ABI Break Switched to using WebAssembly exception handling for C++ errors, Rust panics, and setjmp/longjmp. Projects and build system helpers that have previously set the -fexceptions compilation flag must switch to using -fwasm-exceptions. Furthermore, if a project uses setjmp/longjmp, they must now pass -fwasm-exceptions or -sSUPPORT_LONGJMP=wasm or both at compile time and link time. #5320

  • Fix Fixed a regression in 0.27.1 which caused Pyodide to crash on iPad + Safari. #5695

  • Enhancement Enable WebGL 2 (-sMAX_WEBGL_VERSION=2). WebGL 1 is still available but must be required explicitly (for example, by using OpenGL ES 2.0) #5708

Python API#

  • Enhancement time.sleep() will now stack switch if possible. This allows other events on the event loop to be processed during the sleep. #5686

  • Enhancement Added JsProxy.to_weakref() as a helper method equivalent to WeakRef.new(proxy). Used it to remove a memory leak that occurs when add_event_listener() is used and then the DOM element is removed from JavaScript. #5687

JavaScript API#

  • BREAKING CHANGE When lockfileURL is given to loadPyodide, the base URL for the packages is now calculated from the lockfile URL, not from the indexURL. #5652

  • Enhancement Added support for custom fetchers to pyfetch. #5653

  • Enhancement Property access on a PyProxy of a dictionary will now fall back to __getitem__() if there is no attribute of the given name. #5674

  • Fix Fixes a bug where pyodide.loadPackage() did not respect messageCallback and errorCallback options in some cases. #5692

  • BREAKING CHANGE JavaScript null is now converted to pyodide.ffi.jsnull and not to None. If you want to opt into the old behavior you can pass convertNullToNone: true to loadPyodide(). This compatibility option will be removed in a later release of Pyodide. #5719

  • Enhancement It is now possible to pass null to JavaScript functions and assign it to JavaScript properties via pyodide.ffi.jsnull. #5719

python CLI entrypoint#

  • Enhancement Pyodide now respects the runtime paths of the libraries it loads. Previously, packages that put the libraries in a different directory would not work, but now they will work as long as the -rpath parameter is set correctly during linking. #5610

async function runPythonAsync(code, globals) {
  return await Module.pyodide_py.eval_code_async(code, globals);
